「大学生が使いやすい」を目標に開発したフリマアプリです。とあるインターンで知り合ったインターン生５人にハッカソンに出ようと声をかけ、合計６人で役割を分担して開発しました。開発期間はキックオフから 10 日間です。

私が担当した部分は主に
・ルーティング設定
・プロジェクトの進捗管理
・チャット機能開発
・アイテムカード作成
・アイテム詳細ページ開発
・マイページの出品一覧管理機能
・大学フィルタリング機能　　　　です。

順にログイン → 学内フィルタで同じ大学に登録している人の出品アイテムに絞ることができます。アイテムページ下部の Chat ページボタン押下で Chat ページに遷移できます。Chat ページでは出品者と商品に興味のある人のみが Chat できるようになっています。

このアプリを開発するうえで工夫した点は主に

① データ取得の安定化を意識したコードを書いたこと
➁ チームでの認識をすり合わせるために、できるだけ積極的に交流の場を設けたということ
③ 短い開発期間の中で、必要、重要な機能とそうでない機能に分けて実装したこと
④ 開発経験の少ないチームメンバーとはペアプロ形式で開発したこと

の四点です。

① 開発途中で著しくデータ取得速度が低下したタイミングがありました。データベースへの登録過多が原因だと疑っていましたが、コードを見直すと認証が確認される前にクエリを実行していたことで、Supabase との接続が不安定になっていたということがわかりました。対策として loading 状態を確認し、認証が終了したら useEffect 内で fetch するという流れを作り、安定してデータを取得することができるようになりました。

➁ このハッカソンは東京の 5days のインターンで出会った人たちに私が声をかけて参加しました。６人中４人が地方出身で、かつ関係値がとても深いというわけではなかったこともあり、チーム開発で認識のずれや、コミュニケーションに問題が起きることが予測されました。この問題を解消するために、キックオフ前から事前に Git の勉強会や使う技術の学習など、積極的にスケジュール調整をして交流する機会を設けました。その甲斐もあり潤滑に開発を進めることができたと感じています。

③ 期間が短いということがあり、実装するべき機能とそうでない部分を分けて、チーム内で話し合いました。必要な機能から実装していけたことで最終的にアプリ完成には至らなかったものの機能は各々担当したところを開発することができました。

④ ６人中アプリ開発経験があったのは３人で、チームの半分が開発経験が少ないという状況でした。今回のハッカソンではチームとして開発するということを重要視し、私は未経験のチームメイトとペアプロ形式で開発しました。私に至らない点が多くあったとは思いますが、お互い積極的に交流しながら開発することができ、チームとして開発をすることができたというところがとてもよかったと感じています。
